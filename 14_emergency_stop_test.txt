REM ================================================================
REM  PAYLOAD  : 14 - Emergency Stop (PANIC) Test
REM  VERSION  : 1.0
REM  TARGET OS: Windows (any — uses Notepad)
REM  SAFE     : YES — reads only, designed to be interrupted
REM ----------------------------------------------------------------
REM  PURPOSE:
REM    Tests HAXSTIK's emergency stop functionality.
REM    This payload intentionally runs a slow loop so the operator
REM    can press the HAXSTIK STOP button mid-injection and verify
REM    that injection halts immediately.
REM
REM  HOW TO TEST:
REM    1. Start this payload running
REM    2. Watch the slow countdown typing in Notepad
REM    3. While it is typing (around STEP 3 or 4), press STOP
REM       in the HAXSTIK control panel
REM    4. Verify that typing stops within 1-2 characters
REM    5. Remaining steps should NOT appear in Notepad
REM
REM  WHAT TO LOOK FOR:
REM    - Payload stops typing quickly after STOP is pressed
REM    - No further keystrokes arrive after stop
REM    - HAXSTIK log shows "PAYLOAD STOPPED" or similar
REM    - Notepad is left with partial text (expected)
REM    - Next payload run starts fresh without leftover state
REM
REM  IF STOP DOES NOT WORK:
REM    - Check that STOP button triggers taskSuspend on payload task
REM    - Check FreeRTOS task handle is valid when STOP is called
REM    - Check for missing vTaskDelay() yield points in loop
REM    - If no yield points exist, STOP cannot interrupt — add them
REM ================================================================
