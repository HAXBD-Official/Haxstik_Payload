REM ================================================================
REM  PAYLOAD  : 11 - Keylogger Live Test
REM  VERSION  : 1.0
REM  TARGET OS: Windows / Linux / Mac
REM  SAFE     : YES — opens Notepad, types predictable test strings
REM ----------------------------------------------------------------
REM  PURPOSE:
REM    Tests the HAXSTIK keylogger by typing known strings into
REM    Notepad. Operator enables keylogger from control panel BEFORE
REM    running this payload, then checks if the typed text appears
REM    in the HAXSTIK keylogger screen.
REM
REM  PRE-REQUISITE:
REM    1. Open HAXSTIK OS control panel
REM    2. Go to TOOLS tab
REM    3. Click ENABLE KEYLOGGER (LED dot should turn green)
REM    4. Then run this payload
REM
REM  WHAT TO LOOK FOR:
REM    - Each typed string should appear in the HAXSTIK Keylogger view
REM    - WebSocket dot in keylogger panel should be GREEN (live)
REM    - If keylogger shows nothing: USB CDC connection issue
REM    - If keylogger shows partial text: DEFAULTDELAY may be too fast
REM    - HAXSTIK log: "KEYLOGGER: ON" should be visible
REM
REM  HOW KEYLOGGING WORKS ON HAXSTIK:
REM    HAXSTIK uses USB HID Keyboard LED state changes (CapsLock/NumLock)
REM    to transmit each keystroke back to the device. This is the
REM    "LED timing channel" — no driver needed on target.
REM ================================================================

DEFAULTDELAY 80

GUI r
DELAY 600
STRING notepad
ENTER
DELAY 1200

REM --- Type predictable test strings for keylogger verification ---
STRING KEYLOGGER-TEST-STRING-ALPHA
ENTER
DELAY 200
STRING 1234567890abcdefghijklmnopqrstuvwxyz
ENTER
DELAY 200
STRING THE QUICK BROWN FOX JUMPS OVER THE LAZY DOG
ENTER
DELAY 200
STRING !@#$%^&*()_+-=[]{}|;:,.<>?
ENTER
DELAY 200
STRING username: testuser
ENTER
DELAY 200
STRING password: TestPass123!
ENTER
DELAY 200
STRING KEYLOGGER-TEST-END
ENTER

REM --- Pause to let keylogger buffer flush ---
DELAY 1000
